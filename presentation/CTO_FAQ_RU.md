## БЕЗОПАСНОСТЬ И ДАННЫЕ

### Q: Клиентские данные уходят в OpenAI?
**A:** Нет. В OpenAI отправляется только текст сообщения и системный промпт. Персональные данные клиента (имя, телефон) хранятся исключительно в локальной PostgreSQL базе банка. IP-адреса хранятся в виде SHA-256 хешей (`IP_HASH_SALT`).

### Q: Что если кто-то попытается взломать через чат?
**A:** Четыре уровня защиты:
1. **Domain Guard** — блокирует нерелевантные запросы до GPT
2. **Rate Limiter** — 60 req/мин per IP
3. **Post-filter** — проверяет ответ AI перед отправкой
4. **System prompt isolation** — инструкции изолированы от контента KB

### Q: Как защищён Admin-доступ?
**A:** JWT HS256 токены с 8-часовым сроком. Логин/пароль задаются через env-переменные (`ADMIN_PASSWORD`, `ADMIN_JWT_SECRET`). Пароли не хранятся в открытом виде.

### Q: Можно ли использовать без облака (on-premise)?
**A:** Да. Вся система запускается локально. OpenAI можно заменить на self-hosted модель через любой OpenAI-совместимый API (Ollama, vLLM, LM Studio). Единственное изменение — URL и название модели в конфиге тенанта.

## ТЕХНОЛОГИЯ И АРХИТЕКТУРА

### Q: Почему GPT-4o mini, а не более мощная модель?
**A:** GPT-4o mini оптимален по цена/качество для банковских FAQ: быстрый (< 2с), дешёвый ($0.15/M токенов vs $15/M у GPT-4), достаточно точный для структурированных ответов. Более мощная модель нужна только для сложных финансовых анализов, которые здесь не требуются.

### Q: Что такое RAG и почему это важно?
**A:** RAG (Retrieval-Augmented Generation) — AI не "галлюцинирует", а ищет ответ в базе знаний банка, затем формулирует ответ на основе найденного. Результат: точные ответы по продуктам банка, а не выдуманные цифры.

### Q: Как работает 4-уровневый роутинг?
**A:**
- **FAQ Direct (≥88%)** — точное совпадение, ответ из FAQ без GPT, ~50мс
- **KB Context (50–87%)** — GPT + контекст из базы знаний, 1–2с
- **LLM Only (25–49%)** — GPT без контекста, 1–3с
- **Escalate (<25%)** — горячая линия 1200, мгновенно

### Q: Как масштабируется система?
**A:** Горизонтально через:
- **Multi-tenant архитектура** — один инстанс, много банков
- **Stateless backend** — легко масштабировать через load balancer
- **Connection pooling** в pg — PostgreSQL выдерживает 100+ конкурентных запросов
- **Rate limiting** на уровне IP

### Q: Какой стек? Есть ли зависимость от проприетарного ПО?
**A:** Open-source стек:
- Frontend: React 18, Vite, Tailwind CSS, TypeScript
- Backend: Fastify, Node.js 20, TypeScript
- DB: PostgreSQL 17 + pgvector 0.8.2
- AI: OpenAI API (заменяемо)

## ИНТЕГРАЦИЯ

### Q: Как интегрировать с существующей CRM банка?
**A:** Два варианта:
1. **Webhook** — при создании лида система POST-запросом уведомляет вашу CRM
2. **API polling** — ваша CRM периодически запрашивает новые лиды через REST API

Интерфейс `POST /api/leads` уже существует и принимает: имя, телефон, тип интереса, статус, метка лида.

### Q: Как добавить новые продукты в базу знаний?
**A:** Три способа:
1. **Admin UI** (в разработке) — загрузка через веб-интерфейс
2. **SQL** — прямая запись в таблицу `kb_chunks` с embedding
3. **Scripts** — скрипт `seed-demo.ts` как образец для batch-загрузки

### Q: Как настроить под другой банк?
**A:** В таблице `tenants` задаётся: название, логотип, цвет бренда, языки, hotline, greeting, лимиты. Один SQL-запрос — новый тенант готов.

## ПРОИЗВОДИТЕЛЬНОСТЬ

### Q: Что если OpenAI недоступен?
**A:** Система возвращает fallback-сообщение "Произошла техническая ошибка. Позвоните: 1233." — клиент не видит технической ошибки, получает понятный ответ. Логи сохраняются для диагностики.

### Q: Какова нагрузочная способность?
**A:** Текущая конфигурация выдерживает: ~500–1000 одновременных пользователей (зависит от OAI rate limits). PostgreSQL: 100+ конкурентных запросов. Для больших нагрузок: Redis для rate limiting, replica для DB.

### Q: Как долго хранятся диалоги?
**A:** По умолчанию — бессрочно. Настраивается через политику удаления в БД (`DELETE FROM sessions WHERE created_at < NOW() - INTERVAL 'N days'`). Нет встроенного TTL — это проектное решение для соответствия политике хранения данных банка.

## БИЗНЕС

### Q: Как считается AI Containment Rate?
**A:** `(сессии без эскалации) / (все сессии с ответом AI) × 100`. Текущий расчёт в `/api/admin/intelligence/kpi`. Бенчмарк для банков: 70–85% — хорошо, 90%+ — отлично.

### Q: Как проверить качество ответов AI?
**A:** Три инструмента:
1. **Страница Диалоги** — ручная проверка выборки
2. **Поле `routing_tier`** в сообщениях — видно какой уровень сработал
3. **`confidence` score** — уверенность для каждого ответа

### Q: Сколько стоит API OpenAI в месяц?
**A:** Примерно при 10 000 сообщений/день:
- Input: ~500 токенов × 10K × 30 = 150M токенов → $22.5/мес
- Output: ~200 токенов × 10K × 30 = 60M токенов → $36/мес
- **Итого: ~$58–75/месяц** при 10 000 сообщений в день
