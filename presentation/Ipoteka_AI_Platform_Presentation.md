# СЛАЙД 2
## Проблемы текущего процесса

### Что происходит без платформы:

| Проблема | Последствие | Масштаб |
|----------|-------------|---------|
| **Перегрузка контакт-центра** | Операторы отвечают на одни и те же вопросы по 50–70 раз в день | Рост FTE-затрат |
| **Потеря лидов** | Клиент интересуется ипотекой в 22:00 — некому ответить | Упущенная выручка |
| **Невидимые жалобы** | 8 категорий проблем не отслеживаются системно | Репутационный риск |
| **Отсутствие аналитики** | Руководство не знает, что спрашивают клиенты | Решения без данных |
| **Разрыв между каналами** | Оператор не знает контекст клиента до звонка | Низкое качество сервиса |
| **Нет ночного покрытия** | 18:00–09:00 = мёртвая зона для клиентов | Отток к конкурентам |

### Стоимость каждого из этих пунктов — миллионы сумов в год.

**Технические характеристики:**

**Языковые возможности:**
- 🇺🇿 Узбекский (латиница) — автоопределение языка клиента
- 🇷🇺 Русский — полная поддержка
- Переключение языка в хедере виджета без перезагрузки

**AI-возможности:**
- Стриминг ответов в реальном времени (Server-Sent Events)
- Персона «Премиум Финансовый Консультант Ipoteka Bank»
- Структурированные ответы: эмодзи + заголовки + списки + CTA
- 3 формата ответа: продуктовый, проблемный, рекомендационный
- Домен-гард: отклонение не-банковских вопросов

**Быстрые действия (Quick Actions):**
- Динамические чипы под ответом ИИ

- При покупательском интенте («хочу», «нужна консультация») → лид-чипы
- Аналитика кликов: что нажимают, какой язык, какой интент

**Доверие клиента:**
- «🔒 Ответы носят информационный характер. За официальными решениями обратитесь к специалисту.»
- Нет обещания ставок и условий без актуальной базы знаний

# СЛАЙД 6
## Лидогенерация: от интереса к сделке

### Автоматический захват лидов (реализовано):

**Механизм работы:**
Система анализирует каждое сообщение клиента на 8 сигналов интереса:
ипотека, кредит, вклад/депозит, карта, автокредит, инвестиции, перевод, рассрочка.

При обнаружении сигнала — появляется форма захвата:
```
«Хотите, чтобы специалист связался с Вами?»
Продукт: Кредит

[Ваше имя (необязательно)]
[Номер телефона *: +998 XX XXX XX XX]

[Оставить заявку]  [Позже]
```

**Скоринг лидов 0–100:**

| Фактор | Очки |
|--------|------|
| Указал телефон | +45 |
| Указал имя | +15 |
| Конкретный продукт | +15 |
| Сообщение >40 символов | +10 |
| AI-уверенность >70% | +10 |
| Тип «Обратный звонок» | +5 |
| **Максимум** | **100** |

**Категории лидов:**
- 🔥 **HOT** (90–100): звонить в первую очередь
- ♨️ **WARM** (70–89): звонить в течение дня
- 🧊 **COLD** (<70): стандартная очередь

**5-этапная CRM-воронка:**
```
Новый → Связались → Квалифицирован → Конвертирован → Закрыт
```
Каждый переход: валидация на сервере + запись в историю + тост-уведомление оператору.

# СЛАЙД 8
## Работа с диалогами: контекст до звонка

### Что видит оператор перед звонком клиенту (реализовано):

**Список диалогов:**
- Поиск по тексту сообщений
- Фильтр по языку (Все / 🇺🇿 UZ / 🇷🇺 RU)
- Пагинация по 20 диалогов
- Для каждого: ID сессии, время, язык, последнее сообщение, теги 👥 (лид) / ⚠️ (жалоба)
- Топ-интент: Кредиты / Депозиты / Карты / Мобильный банк / Отделения / Переводы / Прочее

**Детальный просмотр:**
- Полная переписка клиента и ИИ в виде chat-bubble
- Метаданные: язык, количество сообщений, дата начала
- Интент: «🎯 Интерес: Кредиты»
- Теги: 👥 Лид / ⚠️ Жалоба
- Сигналы эскалации помечены оранжевым «⚠ Сигнал жалобы»

**Связь с CRM:**
- Из страницы лидов: кнопка «💬 Открыть диалог» — прямой переход к переписке
- Оператор видит точный момент, когда клиент сообщил интерес
- Нет необходимости расспрашивать клиента заново

### Ценность для операторов:
> «Я звоню клиенту и уже знаю: он интересовался ипотекой, написал 8 сообщений, оставил заявку в 23:15. Разговор начинается с полного контекста, а не с нуля.»

# СЛАЙД 9
## Аналитика жалоб: видимость проблем

### 8 категорий автоматической классификации (реализовано):

| Категория | Иконка | Приоритет | Примеры сигналов |
|-----------|--------|-----------|-----------------|
| Мобильное приложение | 📱 | 🔴 Высокий | «приложение не работает», «ilova ishlamayapti» |
| Карты | 💳 | 🔴 Высокий | «карта заблокирована», «платёж отклонён» |
| Переводы | ↔️ | 🔴 Высокий | «деньги не дошли», «pul yo'qoldi» |
| Авторизация / OTP | 🔐 | 🔴 Высокий | «SMS не приходит», «kod kelmayapti» |
| Кредиты | 🏠 | 🟡 Средний | «кредит отказали», «stavka yuqori» |
| Отделения | 🏢 | 🟡 Средний | «очередь», «грубое обслуживание», «dolgo zhdat» |
| Вклады / Депозиты | 💰 | 🟢 Низкий | «проценты не начислены», «foiz hisoblanmadi» |
| Качество обслуживания | ⭐ | 🟢 Низкий | «ужасный сервис», «shikoyat», «недоволен» |

**Автоматическая эскалация:**
Если за 24 часа накапливается 10+ жалоб одной категории — система автоматически создаёт эскалацию для руководства:
```
🚨 Mobile App: 14 complaints in 24h
Severity: HIGH | Auto-detected: true
Description: 14 сообщений совпали с паттернами «mobile_app»
```

**Тренды vs предыдущий период:**
- Каждая категория: ↑ растёт / ↓ снижается / – стабильно
- Автоматический алерт: «📈 Рост жалоб: Мобильное приложение · Карты»

**Метрики качества AI:**
- **Содержательность (Containment Rate):** % диалогов, решённых без оператора
- **Скорость ответа:** среднее время генерации (мс)
- **Уверенность модели:** средний confidence score
- **Открытые эскалации:** количество нерешённых жалоб

**AI-инсайты (генерируются GPT-4o-mini автоматически):**
Каждый день система анализирует KPI, темы, жалобы и тренды, генерируя 5 управленческих инсайтов:

```
• «Депозиты» стали самым запрашиваемым продуктом — 34% всех диалогов,
  рост +12% по сравнению с прошлой неделей.

• AI решает 87% обращений без участия операторов —
  1,089 клиентов получили помощь без звонка в банк.

• Жалобы на мобильное приложение выросли на 40% —
  требует немедленного внимания IT-команды.

• 38 лидов захвачено за период, из них 5 горячих (HOT).
  Конверсия: 2 сделки закрыто.

• Клиенты на узбекском языке составляют ~45% сессий —
  языковой паритет достигнут.
```
