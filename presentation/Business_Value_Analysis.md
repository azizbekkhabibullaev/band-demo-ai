## 2. РЕШЁННЫЕ БИЗНЕС-ПРОБЛЕМЫ

### 2.1 Перегрузка контакт-центра

**Проблема:** Операторы ежедневно отвечают на одинаковые вопросы о депозитах, кредитах, картах, филиалах, мобильном приложении.

**Решение:** AI берёт на себя стандартные запросы с 4-уровневой маршрутизацией:
- `faq_direct` (≥88% уверенности): прямой ответ из FAQ, <200мс
- `kb_context` (≥50%): ответ на базе базы знаний
- `llm_only` (>25%): генерация GPT-4o-mini с контекстом KB
- `escalate` (<25%): направление к оператору

**Метрика:** Containment Rate = 1 − (эскалации / все диалоги)
**Ожидаемый результат:** ≥80% запросов решаются без оператора

**Расчёт экономии (пример):**
```
Допущения:
- 500 запросов/день × 5 мин/запрос × $0.03/мин оператора
- Текущие затраты: 500 × 5 × $0.03 = $75/день = $27,375/год

При AI Containment Rate 80%:
- AI берёт 400 запросов → экономия $60/день = $21,900/год
- Операторы фокусируются на 100 качественных запросах
```

### 2.2 Потеря лидов во внерабочее время

**Проблема:** Клиенты, интересующиеся ипотекой и кредитами в 21:00–09:00, не получают ответа и уходят к конкурентам.

**Решение:**
- Система обнаруживает интерес к 8 продуктам по ключевым паттернам (ипотека, кредит, вклад, карта, и др.)
- Форма захвата появляется автоматически — имя + телефон +998XXXXXXXXX
- Лид создаётся мгновенно с оценкой 0–100 и типом интереса
- Утром оператор видит очередь лидов, отсортированных по приоритету

**Скоринговая модель:**

| Сигнал | Вес |
|--------|-----|
| Указан телефон | +45 баллов |
| Указано имя | +15 баллов |
| Конкретный продукт | +15 баллов |
| Сообщение >40 символов | +10 баллов |
| AI уверенность >70% | +10 баллов |
| Тип «обратный звонок» | +5 баллов |
| Максимум | 100 баллов |

**HOT-лид** (90–100): телефон + имя + продукт + длинное сообщение = человек готов к сделке.

**Ценность:** Каждый потерянный лид по ипотеке — это потенциально 10 000–50 000 USD комиссии. Один захваченный HOT-лид окупает месяц работы платформы.

### 2.3 Невидимость жалоб

**Проблема:** Банк узнаёт о массовых проблемах (сбой мобильного приложения, проблемы с переводами) из отзывов в App Store или жалоб в регулятор — когда уже поздно.

**Решение:** 8 категорий автоматической классификации жалоб из диалогов клиентов:

| Категория | Приоритет | Примеры сигналов |
|-----------|-----------|-----------------|
| Мобильное приложение | 🔴 Высокий | приложение не работает, crash, зависает |
| Карты | 🔴 Высокий | карта заблокирована, платёж отклонён |
| Переводы | 🔴 Высокий | деньги не дошли, двойное списание |
| OTP / SMS | 🔴 Высокий | код не приходит |
| Кредиты | 🟡 Средний | отказ, высокая ставка |
| Отделения | 🟡 Средний | очередь, грубый персонал |
| Вклады | 🟢 Низкий | проценты не начислены |
| Качество сервиса | 🟢 Низкий | жалоба, недоволен |

**Авто-эскалация:** 10+ жалоб одной категории за 24ч → автоматически создаётся алерт для руководства. До появления публичного скандала.

### 2.4 Отсутствие управленческой аналитики

**Проблема:** Руководство принимает продуктовые решения без данных о реальном спросе клиентов.

**Решение — Dashboard в реальном времени:**
- Топ-5 продуктов по запросам (с трендами vs предыдущий период)
- Уникальные сессии по дням (пики = эффект рекламных кампаний)
- Распределение UZ vs RU (~45%/55%)
- AI-инсайты: GPT-4o-mini анализирует KPI и генерирует 5 управленческих выводов

**Пример AI-инсайта:**
> «Запросы по депозитам выросли на +34% по сравнению с прошлой неделей — возможная реакция на изменение ставок. Рекомендуется усилить продажи через все каналы.»

### 2.5 Разрыв между каналами (оператор не знает контекст)

**Проблема:** Оператор, принимая звонок, начинает с нуля — без знания истории клиента.

**Решение:** Conversation Intelligence:
- Полная история диалога клиента с AI
- Определённый интент (Кредиты / Депозиты / Карты)
- Флаги: есть ли лид, была ли жалоба
- Кнопка «Открыть диалог» прямо из карточки лида

**Ценность:** Оператор тратит 0 секунд на сбор контекста. Конверсия растёт, время разговора сокращается.
